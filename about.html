Job Description

It is the responsibility of the DevOps team at Camis to take into account client reservation windows and customer rush times make provisions to the computational power of dedicated web applications, load balancers, databases, and virtual machines so that interaction with the Camis software system remains unimpeded. This is achieved by the analysis, benchmarking, and testing of each component under expected or actual loads on the system by the quality assurance team, which is then communicated to DevOps to ensure the required cloud computing service plan will be applied using Azure. It is the goal of the DevOps Engineer the ensure that the system can meet the demands of clients and customers, preferably though automated processes, at the best cost efficiency for the company.

To ensure the maximum possible uptime for the Camis 5 application, the DevOps team follows the agile software development approach in creating and maintaining continuous delivery pipelines. A continuous delivery pipeline is a set of steps code changes will follow programmatically to make their way to production. Automated builds, tests, and deployments are orchestrated as one release workflow, and contribute to the next iterative release of the Camis platform. Technologies used to accomplish this include Bitbucket, to track changes to the code base following the Git branching model workflow, Jenkins, to help spread the building and testing of new code changes over multiple service workers, or virtual machines, and Octopus Deploy, to manage releases and automate deployments to production.

A time where I was able to use benchmark testing and a systems thinking approach to the continuous deployment pipeline was for a task that would eliminate an unnecessary unpackaging step within the testing phase of the web application. I was to modify the main existing product pipeline to run end-to-end tests for the client website on a Jenkins build agent before packaging and deployment rather than afterward when the website is unpacked again and hosted through Azure. This was to theoretically save time on the deployment as it would not be packaged first and then unpackaged for the required test suite to run. By eliminating some time on each of the end-to-end test build that Jenkins encountered, the deployment time was decreased as found by the benchmark testing

Following some of the prevalent methodologies known as change control and continuous delivery in DevOps, lean management is another core tool used heavily at Camis. The lean management methodology focuses on small, iterative batches of work, and establishing feedback loops, and visualizations. Camis implements a Kanban board to visually organize existing JIRA tickets at various stages including backlog, in-progress, and review. This lets DevOps team members establish a lean workflow and quickly shift attention between multiple tickets in a single day. Feedback loops are amplified back though the continuous delivery pipeline to provide fast and detailed explanations as to what went wrong for developers to quickly resolve bugs in their code before it reaches production. This is mainly done through automated PowerShell processes created by DevOps to save time and provide transparency for other teams by posting feedback directly on the JIRA ticket.

Splunk Cloud is a cloud-based service and log management tool for ingesting machine data for the creation of automated reports, alerts, and dashboard visualizations. By ingesting Azure SQL, and web application IIS logs, and using the Splunk Search Language, I created multiple reports, alerts, and dashboards for the IT Systems team to monitor web traffic and database queries to easily visualize suspicious activity on the website or possible SQL injection attacks. By graphing the most out-of-date versions for each web browser per IP address in the past 6 months on client websites, I could quickly provide the information requested by the IT Systems team, bridging the gap between Dev and Ops.
